1安裝 Claude Code

約 10 分鐘

Claude Code 是 Anthropic 官方的 AI 工作工具，這套系統靠它運轉。

  1. 打開你電腦上任何一個網頁瀏覽器（Chrome、Safari、Edge 都可以），在網址列輸入 claude.ai/code（或搜尋「Claude Code 安裝」找官方頁面）
  2. 照官方指示安裝（Mac 與 Windows 都有官方安裝器）
  3. 安裝完成後，用你的 Claude 帳號登入

卡住了？官方安裝頁有中文說明；或先跳到本頁最下方「常見問題」。

2放好這個包

約 5 分鐘

  1. 打開檔案管理程式（Mac 叫「Finder」、Windows 叫「檔案總管」），找到你下載的 core-pack 資料夾
  2. 在資料夾上按右鍵 → 複製，貼到「文件」底下，然後右鍵 → 重新命名，改名為「我的AI內容引擎」
  3. 記住這個位置，以後所有工作都在這個資料夾裡發生

3啟動

約 2 分鐘

  1. 打開「終端機」（Mac：按鍵盤 ⌘＋空白鍵，輸入 Terminal，按 Enter；Windows：按開始鍵，輸入 PowerShell，按 Enter）
  2. 你會看到一個黑色（或白色）的文字視窗——這是正常的，不用緊張，接下來只需要打兩行字
  3. 輸入 cd （cd 後面有一個空格），然後用滑鼠把「我的AI內容引擎」資料夾按住、拖曳到這個文字視窗上放開（路徑會自動出現），按 Enter
  4. 輸入 claude，按 Enter——看到 Claude 跟你打招呼，就成功了

4安裝精靈

約 15 分鐘

對 Claude 說：

「開始安裝」

它會照精靈指示，一題一題問你，然後自動把答案填進系統檔案。你只要像聊天一樣回答。

給 AI 的安裝精靈指示（技術細節，不用自己看懂）
順序問題注意事項
1署名要用什麼名字？—
2你的年齡？會成為文章的身份標記，真實才有力量
3一句話介紹自己（年齡＋反差＋具體事）幫用戶打磨到三要素齊備才收下
4真實可驗證的實績數字只填真的，答不出來就填「暫無，累積中」
5固定結語怎麼寫？定了就不再改，是品牌資產
6有電子報或部落格嗎？沒有就整段留空，之後可補

5寫下你的故事素材

約 30 分鐘，最重要的一步

AI 寫出來的文章好不好，九成取決於這一步。

對 Claude 說：

「幫我建故事素材」

Claude 會像採訪一樣問你 3–5 個真實經歷（一次失敗、一次轉折、一個具體數字的時刻、一個讓你改變做法的瞬間），整理後存進 stories/我的故事素材.md。

只講真的。這套系統的規則是：寧可標「待補」，不編造。

6跑通第一條流程

約 20 分鐘

  1. 對 Claude 說：
    「寫長文：我為什麼開始用 AI」（或你自己的主題）
  2. Claude 會先給你段落大綱 → 你說 OK → 產出一篇 2000 字以上的長文草稿
  3. 對 Claude 說：
    「送審」
    → 系統自動跑三關審核，給你通過或打回的報告
  4. 打回就照報告修（可以請 Claude 指出要改哪裡，但改不改、怎麼改由你決定），修完再說「送審」

7切串文＋發布

約 10 分鐘

過審後，對 Claude 說：

「切串文」

它會把長文切成每則 500 字以內的 Threads 串文（第 1/N 格式，結語在最後一則）。

然後打開手機 Threads：

  1. 貼出第 1 則（一般發文）
  2. 發出後，點開自己剛發的這則貼文，按下方的「回覆」按鈕，貼上第 2 則，發出
  3. 之後每一則都是「回覆上一則」——像自己接自己的話，讀者就會看到一串連續的內容
  4. 全部貼完，完成

發布永遠由你的手按下去。 這是系統的設計，不是限制——AI 永遠碰不到你的帳號。

8手機回報（選配，強烈建議）

約 10 分鐘

裝了這一步，AI 完成工作（草稿寫完、審核通過或打回）會直接傳訊息到你的手機 Telegram——你人在外面談生意，也知道系統跑到哪了。

沒用過 Telegram？它是一個免費通訊 App（類似 LINE）。這一步完全可以先跳過，系統其他功能不受影響。

  1. 手機安裝 Telegram（免費，App Store / Google Play 搜尋）
  2. 在 Telegram 搜尋 @BotFather（官方機器人，有藍勾勾）→ 傳 /newbot → 照提示取名字 → 拿到一串 token（長得像 123456:ABC-DEF...），複製起來
  3. 對 Claude 說：
    「設定手機回報」
    ，把 token 貼給它——精靈會完成剩下的事
給 AI 的手機回報設定指示（技術細節）

向用戶要 BotFather 給的 token → 呼叫 getUpdates 取得 chat id → 建立 config/tg.json → 複製 tg-notify-template.py 為 tg-notify.py → 發一則測試訊息確認成功。提醒用戶 token 不要分享、不要截圖給別人。

常見問題

Q：我完全不會程式，真的裝得起來嗎？
全程只有兩個終端機動作：cd＋拖資料夾、輸入 claude。其餘都是「對 AI 說話」。這套包的設計標準是：照著做就能通，卡住的每一步都算我們的缺陷，回報給我們。
Q：換電腦怎麼辦？
整個資料夾複製過去，重裝 Claude Code，繼續用。你的故事素材、判例庫、所有文章都在資料夾裡。
Q：AI 會不會亂編我的經歷？
系統規則明文禁止：缺資料標「⬜ 待補」，不自行補全。而且發布前你會親眼審過每一個字——發布權永遠在你手上。
Q：為什麼不能自動發文？
可以，那是進階模組（見 scripts-manifest.md），需要申請 Meta 開發者帳號。我們刻意不放在第一天：先讓系統跑起來、你先發出前幾篇，再決定要不要自動化。
Q：卡住了怎麼辦？（SOS 三步）
1. 先問 Claude：把你卡住的畫面或錯誤訊息直接描述給它，多數問題它能當場帶你解決。
2. 重開一次：關掉終端機視窗，照第 3 步重來一遍。
3. 還是不行：記下你卡在第幾步＋畫面上的文字，回信 allenchiu0903@gmail.com。你卡住的每一步都算我們的缺陷，不是你的問題。
